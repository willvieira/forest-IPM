Create `.github/workflows/` directory and write `R-CMD-check.yaml`.

Base this on the r-lib/actions check-release template (https://github.com/r-lib/actions/blob/v2/examples/check-release.yaml) with these locked modifications:
- Linux only: `runs-on: ubuntu-latest` (no matrix, no macOS, no Windows rows)
- R release only: `r-version: release` in setup-r (no oldrel, no devel)
- Triggers: `push` + `pull_request` on branches `[main, master]`
- `permissions: read-all` at workflow level

Final YAML structure:

```yaml
on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

name: R-CMD-check.yaml

permissions: read-all

jobs:
  R-CMD-check:
    runs-on: ubuntu-latest
    env:
      GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}
      R_KEEP_PKG_SOURCE: yes
    steps:
      - uses: actions/checkout@v4
      - uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true
      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: any::rcmdcheck
          needs: check
      - uses: r-lib/actions/check-r-package@v2
        with:
          upload-snapshots: true
          build_args: 'c("--no-manual", "--compact-vignettes=gs+qpdf")'
```

Note: `setup-r-dependencies` automatically detects system dependencies for Rcpp/RcppEigen on Ubuntu via pak — no manual `apt-get install` step needed. Only add the explicit fallback `sudo apt-get install -y build-essential` if the first CI run fails with `g++: not found` or `Eigen/Core: No such file`.

Write `.github/workflows/test-coverage.yaml`.

Base this on the r-lib/actions test-coverage template (https://github.com/r-lib/actions/blob/v2/examples/test-coverage.yaml) with locked modifications (Linux only, R release only). This is a separate workflow file from R-CMD-check.yaml so that coverage failures do not block the check result (and vice versa).

Final YAML structure:

```yaml
on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]

name: test-coverage.yaml

permissions: read-all

jobs:
  test-coverage:
    runs-on: ubuntu-latest
    env:
      GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}
    steps:
      - uses: actions/checkout@v4
      - uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true
      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: any::covr, any::xml2
          needs: coverage
      - name: Test coverage
        run: |
          cov <- covr::package_coverage(
            quiet = FALSE,
            clean = FALSE,
            install_path = file.path(normalizePath(Sys.getenv("RUNNER_TEMP"), winslash = "/"), "package")
          )
          covr::to_cobertura(cov)
        shell: Rscript {0}
      - uses: codecov/codecov-action@v5
        with:
          fail_ci_if_error: ${{ github.event_name != 'pull_request' && true || false }}
          file: ./cobertura.xml
          plugin: noop
          disable_search: true
          token: ${{ secrets.CODECOV_TOKEN }}
```

Notes:
- `secrets.CODECOV_TOKEN` is optional for public repos — codecov.io will accept uploads without it, but the secret should be added to GitHub repo Settings > Secrets > Actions after the user activates the repo on codecov.io
- C++ coverage (src/eigen.cpp) is NOT measured by covr — this is expected behavior; the 70% target applies to R code only
- `xml2` is paired with `covr` to produce the Cobertura XML format required by codecov-action@v5
