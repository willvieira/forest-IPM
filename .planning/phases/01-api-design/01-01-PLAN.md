**1. Overview and Design Principles**
- Package description: "A stochastic dynamic forest Integral Projection Model built on Bayesian species-specific
  demographic rates with endogenous competition and flexible climate drivers"
- Five principles: (a) fail-fast validation at construction time, (b) immutable stand objects,
  (c) stochasticity fully resolved in parameters() — everything downstream is deterministic,
  (d) single unified class for single- and multi-species models, (e) no wrapper functions —
  researchers compose constructors + engines directly

**2. Canonical Workflow**
Show the full workflow example exactly as in CONTEXT.md:
```r
stand <- stand(data)
mod   <- species_model(stand)
pars  <- parameters(mod, draw = "random", seed = 42)
env   <- env_condition(MAT = 8, MAP = 1200)
ctrl  <- control(years = 100, delta_time = 1)
lambda(mod, pars, stand, env)
project(mod, pars, stand, env, ctrl)
```
Include a note: "API-03 originally specified a single `run_ipm()` entry point. The design session
replaced this with the five-constructor + two-engine pattern above. This canonical workflow IS
the fulfillment of API-03."

**3. Exported Functions Reference**

For each function include: signature, arguments table (name | type | default | description),
return value (type + structure), validation rules (exact error conditions), and a usage example.

*3.1 stand(data)*
- Signature: `stand(data)`
- `data`: data.frame or tibble; must contain EITHER `size` OR `dbh` column (numeric, > 0, mm),
  EITHER `sp` OR `species` OR `species_id` column (character, must match known IDs), AND `plot_size`
  column (numeric scalar, m2). Note: plot_size is a column in data, not a separate argument.
- Returns: object of S3 class `"ipm_stand"` with fields:
  - `$trees`: data.frame of individual-level records (standardized column names: size_mm, species_id)
  - `$species`: character vector of unique species IDs in the stand
  - `$plot_size`: numeric scalar, plot area in m2
- Validation (all checked at construction; cli_abort() on failure):
  - data has no required individual size column: "data must contain a column named {.field size} or {.field dbh} (size in mm)."
  - size or dbh <= 0: "All tree sizes must be positive (> 0 mm). Found {n} non-positive values."
  - data has no species column: "data must contain a column named {.field sp}, {.field species} or {.field species_id}."
  - data has no plot_size column: "data must contain a {.field plot_size} column (plot area in m2)."
- stand is immutable: never modified in-place by any function

*3.2 species_model(x, on_missing = c("error", "drop", "static"))*
- Signature: `species_model(x, on_missing = "error")`
- `x`: character vector of species IDs, OR an `ipm_stand` object (species inferred from stand$species)
- `on_missing`: behavior when a species ID is not in `supported_species()`. Options:
  - "error" (default): cli_abort() with closest-match suggestion
  - "drop": silently remove species from model (and from stand if stand was provided)
  - "static": keep species for competition background only; exclude from lambda()/project() and trees stay static over time
- Returns: object of S3 class `"ipm_spModel"` with fields:
  - `$species`: character vector of species IDs in the model
  - `$on_missing`: the on_missing value used at construction
- Validation on_missing = "error" error template:
  cli_abort(c(
    "{.arg x} contains species IDs not found in {.run supported_species()}.",
    "x" = "Unknown: {.val {bad_ids}}.",
    "i" = "Did you mean {.val {closest_match}}?",
    "i" = "Run {.run supported_species()} to see all valid IDs."
  ))
- Note: use stringdist::amatch() for closest-match suggestion, not a hand-rolled Levenshtein

*3.3 parameters(mod, draw = "random", seed = NULL)*
- Signature: `parameters(mod, draw = "random", seed = NULL)`
- `mod`: object of class `"ipm_spModel"`
- `draw`: one of "mean" (posterior mean), "random" (random draw), or a positive integer
  1-1000 (select exact posterior draw index). Integer draw = 1 means draw #1, NOT "draw 1 replicate".
- `seed`: integer or NULL; used only when draw = "random"; controls reproducibility
- Returns: object of S3 class `"ipm_parameters"` with fields:
  - `$species_params`: named list keyed by species_id, each containing $fixed, $random_effects, $draw_id
  - `$draw_type`: the resolved draw specification
  - `$seed`: the seed used (NULL if draw was not "random")
- All stochasticity is resolved here. Everything downstream of parameters() is deterministic.
- draw argument forms (all three must be documented with examples):
  parameters(mod, draw = "mean")               # posterior mean
  parameters(mod, draw = "random", seed = 42)  # reproducible random draw
  parameters(mod, draw = 314)                  # exact draw #314 from posterior

*3.4 env_condition(MAT, MAP)*
- Canonical name is `env_condition()` (singular). NEVER use `env_conditions()` plural.
- Signature: `env_condition(MAT, MAP)`
- `MAT`: Mean Annual Temperature in Celsius; numeric scalar OR a function of time function(t) ...
- `MAP`: Mean Annual Precipitation in mm/year; numeric scalar OR a function of time function(t) ...
- MAT/MAP scaling to [0,1] applied internally using stored scaling parameters. Scaling parameters
  are NEVER exposed in the returned object or to the user.
- Returns: object of S3 class `"ipm_env"` with fields:
  - `$MAT`: the MAT value or function as provided
  - `$MAP`: the MAP value or function as provided
- Usage examples:
  env_static  <- env_condition(MAT = 8, MAP = 1200)
  env_dynamic <- env_condition(MAT = function(t) 8 + 0.02 * t, MAP = 1200)

*3.5 control(years = 100, delta_time = 1, store_every = 1)*
- Signature: `control(years = 100, delta_time = 1, store_every = 1, bin_width = 1)`
- `years`: positive integer; total simulation timesteps
- `delta_time`: positive numeric; timestep length in years (default 1 = annual)
- `store_every`: positive integer; store stand state every N timesteps. Default 1 = store all.
  Use store_every = 10 for runs > 200 years to reduce memory.
- `bin_width`: positive integer; bin width used in the kernel discretization
- Returns: object of S3 class `"ipm_control"`

*3.6 lambda(mod, pars, stand, env, ctrl)*
- Signature: `lambda(mod, pars, stand, env, ctrl)`
- `mod`: "ipm_spModel" — defines which species to compute lambda for
- `pars`: "ipm_parameters" — must contain parameters for all species in mod
- `stand`: "ipm_stand" — provides initial size distribution and competition context
- `env`: "ipm_env" — climate drivers
- `ctrl`: "ipm_control" — simulation configuration. `years` and `store_every` are not needed here and only used in the `project` engine
- Returns: object of S3 class `"ipm_lambda"` which is a named numeric vector, one element per species in mod.
  Example: lam <- lambda(mod, pars, stand, env)  # c(ABIBAL = 1.03, ACERUB = 0.97)
- lambda() does NOT require parameters for heterospecific species; competition computed from
  stand structure only. No dynamic coupling.
- Validation: pars must cover all species in mod; stand species must be in mod (cli_abort if not)
- Because the natural workflow will be to set `stand`, then `species` and then `parameters`, all species in `mod` will have their parameters meaning that `lambda` will compute lambda for all species. In this case, it's important to specify to the user that, if they want lambda for only a focal species, it's better to manually define the species_id in the `paramater()` constructor this way lambda only computes for the species with available parameters and saves lots of expensive lambda computation time

*3.7 project(mod, pars, stand, env, ctrl)*
- Signature: `project(mod, pars, stand, env, ctrl)`
- `mod`, `pars`, `stand`, `env`, `ctrl`: same as lambda()
- Returns: object of S3 class `"ipm_projection"` (see Section 4 for full structure)
- With multiple species in mod: fully coupled dynamics — competition updated each timestep
- With single species in mod: independent dynamic
- Validation: same as lambda()

*3.8 supported_species()*
- Signature: `supported_species()`
- No arguments
- Returns: tibble with exactly three columns:
  - species_id (character): the ID string used throughout the API (e.g., "ABIBAL", "ACERUB")
  - common_name (character): English common name (e.g., "Balsam fir")
  - nom_commun (character): French common name (e.g., "Sapin baumier")
  - growth_model (character): model string (e.g., "intcpt_plot_comp_clim")
  - surv_model (character): model string (e.g., "intcpt_plot_comp_clim")
  - recruit_model (character): model string (e.g., "intcpt_plot_comp_clim")
- One row per species

**4. S3 Classes**

For each class: class name string, constructor pattern, fields, print method contract.

*4.1 ipm_stand*: class "ipm_stand"; uses constructor/validator/helper layers (see Section 6)
*4.2 ipm_spModel*: class "ipm_spModel"; single and multi-species share this class (NEVER "ipm_single_model")
*4.3 ipm_parameters*: class "ipm_parameters"
*4.4 ipm_env*: class "ipm_env" (returned by env_condition())
*4.5 ipm_control*: class "ipm_control"
*4.6 ipm_projection*: class "ipm_projection"; complete field specification:
  - $species: character vector of species in projection
  - $years: numeric vector of timesteps (length = ceiling(ctrl$years / ctrl$store_every))
  - $lambda: named list; one element per species, each a numeric vector over stored timesteps
  - $stand_series: list of ipm_stand objects, one per stored timestep
  - $summary: tibble; columns: timestep (int), species_id (chr), lambda (dbl), n_trees (int)

S3 method conventions — each class must implement:
- print.<class>(): one-liner: class name + key dimensions + first 3 species
- summary.<class>(): detailed structured output
- plot.ipm_projection(): trajectory plot (lambda and/or tree count over time, per species)

**5. Naming Conventions**

- Exported functions: snake_case (stand, species_model, parameters, env_condition, control,
  lambda, project, supported_species)
- S3 class names: "ipm_" prefix + noun (ipm_stand, ipm_spModel, ipm_parameters, ipm_env,
  ipm_control, ipm_projection)
- S3 methods: <generic>.<class> standard pattern (print.ipm_stand, summary.ipm_spModel,
  plot.ipm_projection)
- Low-level constructors (internal, not exported): new_<class> (new_ipm_stand, new_ipm_spModel, ...)
- Validators (internal): validate_<class> (validate_ipm_stand, ...)
- Internal helper functions: dot prefix (.scale_climate, .compute_competition, .validate_stand_data)
- Internal checks: always use inherits(x, "ipm_stand") — never inherits(x, "stand")

**6. Constructor/Validator/Helper Pattern**

Document the three-layer S3 pattern that ALL constructors must follow (Advanced R, Wickham):
1. new_<class>(...) — low-level, fast, minimal checks, sets class and attributes (INTERNAL)
2. validate_<class>(x) — thorough value checks, throws cli_abort() on failure (INTERNAL)
3. <helper>(...) — user-facing exported function; calls constructor + validator (EXPORTED)

Include the ipm_stand example:

```r
# Low-level constructor (internal)
new_ipm_stand <- function(trees, species, plot_size) {
  structure(
    list(trees = trees, species = species, plot_size = plot_size),
    class = "ipm_stand"
  )
}

# Validator (internal)
validate_ipm_stand <- function(x) {
  # check required columns, positive sizes, known species
  x
}

# User-facing helper (exported)
stand <- function(data) {
  validate_ipm_stand(
    new_ipm_stand(
      trees = data,
      species = unique(data$species_id),
      plot_size = data$plot_size[[1]]
    )
  )
}
```

**7. Validation Rules and Error Messages**

Table: constructor | condition | cli_abort() message template

All cli_abort() calls must use the named vector format:
cli_abort(c(
  "Primary message.",
  "x" = "What went wrong.",
  "i" = "How to fix it."
))

Formatting tokens: {.arg arg_name}, {.field col_name}, {.val value}, {.run fn()}.

Constructor-specific rules:
- stand(): missing size/dbh column, non-positive sizes, missing species column, missing plot_size column
- species_model(): unknown species IDs (with closest-match suggestion via stringdist)
- parameters(): invalid draw argument type, draw integer out of 1-1000 range, mod not ipm_spModel
- env_condition(): MAT/MAP not numeric scalar or function
- control(): years not positive integer, delta_time not positive numeric, store_every not positive integer, bin_width not positive integer

Cross-constructor rules at engine call time:
- lambda()/project(): pars does not cover all species in mod:
  "pars does not contain parameters for all species in mod. Missing: {.val {missing_ids}}."
- lambda()/project(): stand contains species not in mod:
  "stand contains species not in mod: {.val {extra_ids}}."
- Cloud unreachable (in parameters() during data fetch):
  "Could not reach cloud host. Check your internet connection or use a previously cached species_model."

**9. Design Decisions and Rationale**

- Five constructors + two engines instead of run_ipm(): composability; researchers can call
  lambda() without projection; parameters() cleanly separates stochasticity from determinism.
  API-03 originally specified run_ipm(species_id, lat, lon, climate, n_draws = 1, ...).
  The canonical workflow (Section 2) supersedes and fulfills API-03.
- One ipm_spModel class for single and multi-species: avoids API branching; multi-species is
  a special case handled by the same constructors and engines, not a separate concept.
- Fail-fast validation at construction time: error messages point to the exact source;
  engines (lambda, project) are clean — they receive valid objects and can trust them.
- S3 over R6/S4: simpler, idiomatic for modeling packages; immutability of stand enforced
  by convention (not reference semantics), consistent with tidyverse ecosystem.
- env_condition() (singular): chosen over env_conditions() to avoid implying plural/list
  semantics; the function creates one environment condition object.

**10. Versioning and Extension Points**

Designed extension points for future milestones (do NOT implement in v1):
- v2 — Spatial random effects: parameters(mod, draw = 12, random_effect = spatial_re(lat, lon))
- Future — Climate lookup helper: get_climate(lat, lon) — deferred, requires spatial database
- v3+ — Multi-species community extensions: out of scope for v1 API
