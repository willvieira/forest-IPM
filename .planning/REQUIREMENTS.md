# Requirements: forestIPM

**Defined:** 2026-02-25
**Core Value:** Any researcher can run a full IPM for any supported species at any location by installing the package — no local data setup, no manual sourcing, no cluster required.

## v1 Requirements

Requirements for initial release. Covers: installable package + code cleanup + cloud parameter access.

### API Design

- [ ] **API-01**: All exported functions listed with finalized signatures, argument names, and return types
- [ ] **API-02**: `supported_species()` — returns data frame of available species IDs, names, and model variants
- [ ] **API-03**: `run_ipm(species_id, lat, lon, climate, n_draws = 1, ...)` — single entry point wrapping get_params → init_pop → mkKernel → project; signature finalized before implementation
- [ ] **API-04**: Naming conventions documented and applied consistently across all exported functions
- [ ] **API-05**: API design document produced capturing decisions, rationale, and interface contracts for Phase 2 implementation

### Package Infrastructure

- [ ] **PKG-01**: Package installs successfully via `devtools::install_github("wvieira/forest-IPM")` on a fresh R environment
- [ ] **PKG-02**: `DESCRIPTION` file declares all dependencies with version constraints (Imports, Suggests, LinkingTo)
- [ ] **PKG-03**: `NAMESPACE` file generated by roxygen2 with all `@importFrom` declarations covering tidyverse symbols used in `R/params.R` (map, filter, pivot_wider, pivot_longer, reframe, select, mutate, str_replace, case_match, set_names, `%>%`)
- [ ] **PKG-04**: Rcpp C++ code compiles successfully — `devtools::load_all()` runs without errors
- [ ] **PKG-05**: `R CMD check` passes with 0 errors and 0 warnings

### Bug Fixes

- [ ] **BUG-01**: `src/eigen.cpp` eigenvalue solver fixed — `SelfAdjointEigenSolver` (assumes symmetric input) replaced with `EigenSolver` (general non-symmetric matrices), since IPM kernel K = P + F is not symmetric
- [ ] **BUG-02**: `exists('fct')` global state check in `init_pop()` (kernel.R) removed or replaced with a proper function argument
- [ ] **BUG-03**: Deprecated `purrr::as_vector()` calls in `params.R` replaced with `unlist()` to prevent breakage on purrr >= 1.0.0

### Remote Data Layer

- [ ] **DATA-01**: All species parameter files (31 species × 3 model variants) converted from local RDS to Parquet format with Hive partitioning by `species_id` and `vital_rate`
- [ ] **DATA-02**: Parquet dataset uploaded to a cloud host verified to support HTTP 206 Partial Content range requests (verified via `curl --range` test before upload)
- [ ] **DATA-03**: `get_params(species_id, model = "intcpt_plot_comp_clim", method = "mean")` function fetches only the requested species parameters from cloud via Arrow predicate pushdown — no full dataset download
- [ ] **DATA-04**: Local parameter cache implemented — repeated calls to `get_params()` for the same species read from local disk, not cloud
- [ ] **DATA-05**: Cache directory uses `tools::R_user_dir("forestIPM", "cache")` (user-local, survives package reinstalls)
- [ ] **DATA-06**: Bundled offline mini-dataset included in package (`sysdata.rda`) — posterior means for 2 representative species enabling examples and tests without network access

### Documentation and Tests (v1 baseline)

- [ ] **DOC-01**: All exported functions have roxygen2 documentation with `@param`, `@return`, and `@examples` (examples use bundled mini-dataset, not live network)
- [ ] **DOC-02**: testthat test suite covering PKG, BUG, and DATA requirements using synthetic fixtures — CI passes without network access

## Out of Scope

| Feature | Reason |
|---------|--------|
| Fitting new species to inventory data | This package consumes posteriors, not fits them — separate modeling pipeline |
| Real-time climate data fetching | User provides climate inputs; avoids external API dependency |
| GUI or Shiny app | Programmatic R API only; web interface is a separate product |
| Windows binary support initially | C++ Rcpp compilation targets Linux/macOS first; Windows deferred |
| Multi-species community IPM (v1) | Single-species IPM first; coexistence adds community-kernel complexity |
